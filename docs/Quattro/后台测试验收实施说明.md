# 后台测试验收实施说明

## 默认运行方式

`tools/run-tests.ps1` 调用的启动型验收通过 `tools/QuattroTestHarness.ps1` 启动 Quattro 子进程。每个进程使用唯一临时运行根目录，并向子进程传递：

- `QUATTRO_USER_CONFIG_DIR`：测试专用配置、数据库和日志目录；
- `QUATTRO_TEST_MODE=1`：启用测试运行根目录校验；
- `QUATTRO_TEST_NO_FOCUS=1`：禁止公共激活入口抢前台焦点；
- `QUATTRO_ACCEPTANCE_MODE=background`：窗口显示后降到用户程序之后；
- `QUATTRO_TEST_RUN_ID`：测试运行标识。

测试根目录必须包含 `.quattro-test-root`。Quattro 在测试模式下找不到该标记时拒绝启动，避免测试错误回落到正式用户目录。

测试清理只允许使用本次启动返回的 PID。禁止使用按进程名查找并结束所有 `Quattro` 的清理方式。

## 窗口与焦点

后台验收窗口保持可见、非最小化、非激活，并使用 `HWND_BOTTOM` 排到用户程序之后。公共主题窗口在测试模式下增加 `WS_EX_NOACTIVATE`，主窗口和主题对话框统一使用公共后台策略。

测试 harness 会在窗口出现时验证测试 PID 没有成为前台窗口；违反时验收直接失败。

自动化验收不调用 `SetForegroundWindow`、`SetActiveWindow`、`SetFocus`，也不临时切换 `HWND_TOPMOST`。验收宿主窗口本身同样使用 `WS_EX_NOACTIVATE`、`SW_SHOWNOACTIVATE` 和 `HWND_BOTTOM`，避免验收程序抢占用户正在使用的窗口、输入法或任务栏注意状态。

## 鼠标与交互状态

后台自动化验收不读取、移动或点击真实鼠标，不使用 `GetCursorPos`、`SetCursorPos`、`SendInput`、`mouse_event` 等桌面输入接口。

- Tooltip 视觉状态由验收进程直接调用公共 `ThemedUi::ShowTooltip`，传入固定屏幕坐标，并在 100%、125%、150% DPI 下分别捕获独立 Tooltip HWND；
- 控件 hover 与 Tooltip 绑定行为由单元测试向控件发送固定客户区坐标的 `WM_MOUSEMOVE`，并通过测试 `ThemedTooltipRegistry` 验证显示次数、文本和隐藏行为；
- Table hover 等进程内状态使用固定客户区坐标消息触发，不依赖桌面光标当前所在位置。

真实鼠标、全局键盘、系统托盘和桌面合成关系只能在显式标记的交互式视觉验收中使用；该类脚本默认拒绝运行，并要求独立、无人使用的 Windows 交互会话。

## 截图

普通窗口、对话框和 Tooltip 使用各自目标 HWND 的 `PrintWindow` 捕获；原生 popup 菜单通过独立的 `#32768` HWND 捕获。不使用桌面 `CopyFromScreen` 或屏幕 `BitBlt` 作为后台验收回退。这样即使被 IDE 或浏览器覆盖，截图仍只来自目标窗口。

`PrintWindow` 返回成功后仍必须检查位图是否包含有效视觉内容，避免把黑图、纯色图或未绘制位图误判为成功。如果目标窗口无法被 HWND 捕获，验收应失败并明确记录捕获方式，后续增加进程内离屏绘制或专用验收通道；不得静默保存被其他程序覆盖的桌面像素，也不得通过激活、置顶窗口规避失败。

依赖真实前台、系统托盘、真实鼠标或桌面合成关系的场景仍应放到独立 Windows 测试会话或 VM 中，不属于 `BackgroundSafe` 默认套件。

## 就绪时序

Quattro 测试模式在主窗口发布单实例句柄后写入 `logs/test-ready.txt`。启动型验收必须等待该标记后再发送命令，避免在 `MainWindow::Create()` 尚未完成时提前销毁窗口或发送消息。
